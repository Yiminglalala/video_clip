## 一、ACRCloud 编码 Bug 修复

### 问题描述
ACRCloud API 返回 `Content-Type: text/plain`（无 charset 声明），Python `requests` 库的 `resp.json()` 会按 ISO-8859-1 解码响应体，导致中文歌名/歌手名全部乱码（如 `é™ˆæ¥šç"Ÿ` 之类）。

### 修复内容

| 文件 | 行号 | 修改 |
|------|------|------|
| `src/lyric_subtitle.py` | L400 | `resp.json()` → `json.loads(resp.content.decode("utf-8"))` |
| `src/lyric_subtitle.py` | L492 | `resp.json() if resp.text else {}` → `json.loads(resp.content.decode("utf-8")) if resp.content else {}` |

### 验证结果
- ✅ 修复后不再乱码，中文消息正确显示（如 `msg=invalid signature`）
- ⚠️ 当前凭据返回 `invalid signature`，需用户去 ACRCloud 控制台核实 Secret

## 五、Tab1 🎬 视频切片 — 功能逻辑 & 模型清单

### 5.1 四步工作流

| 步骤 | 函数 | 功能 |
|------|------|------|
| Step 0: 选择视频 | `_render_step0_select_video()` | 上传/粘贴路径 → 视频预览 → 歌手名/演唱会名 → 高级设置 |
| Step 1: 处理视频 | `_render_step1_processing()` | 调用 `LiveVideoProcessor.process_video()` 执行全流程 |
| Step 2: 预览编辑 | `_render_step2_preview_edit()` | 视频播放+时间轴跳转 → 片段标签编辑 → 起止时间微调 |
| Step 3: 导出视频 | `_render_step3_export()` | 选格式/质量 → FFmpeg 逐段切片导出 |

### 5.2 核心处理流程

```
输入视频
  │
  ├─ 1. 提取音频 (FFmpeg → WAV 22050Hz)
  │
  ├─ 2. 三路边界检测（确定歌曲边界）
  │    ├─ OCR 标题画面检测 (TextFrameDetector)     ← 权重 0.60
  │    ├─ 音频能量谷值                               ← 已禁用（误判太多）
  │    └─ MSAF 音乐结构分析 (Foote/SF/OLDA)         ← 权重 0.15
  │
  ├─ 3. 加权聚类融合 → 歌曲边界
  │
  ├─ 4. 逐歌处理 (_create_songs)
  │    ├─ Demucs 人声分离 (htdemucs_ft)
  │    ├─ SongFormer 段落分类 (8类)
  │    ├─ FireRedAED singing/speech 修正
  │    ├─ 识曲 (ShazamIO → ShazamAPI)
  │    └─ OCR 歌名提取 + 歌词识曲
  │
  ├─ 5. 片段时长约束 (8-20s)
  │
  └─ 6. FFmpeg 切片导出 + 可选豆包字幕烧录
```

### 5.3 使用的模型/服务

| 模型/服务 | 用途 | 来源 | GPU/CPU | 配置项 |
|-----------|------|------|---------|--------|
| **SongFormer** (99.6MB) | 音乐段落分类（8类：前奏/主歌/副歌/间奏/独奏/观众/讲话/尾奏） | `SongFormer_install/ckpts/SongFormer.safetensors` | GPU | `enable_songformer` |
| **MuQ-large** | 音频嵌入提取 → SongFormer 输入特征 | HuggingFace `OpenMuQ/MuQ-large-msd-iter` | GPU | SongFormer 内部 |
| **MusicFM25Hz** (1.3GB) | 音频嵌入提取 → SongFormer 输入特征 | `SongFormer_install/ckpts/MusicFM/pretrained_msd.pt` | GPU | SongFormer 内部 |
| **Demucs htdemucs_ft** | 人声/伴奏分离 | `demucs` pip | GPU | `enable_demucs` |
| **FireRedVAD** | 语音活动检测 | `pretrained_models/xukaituo/FireRedVAD/VAD/` | GPU | AudioAnalyzer.vad |
| **FireRedAED** | speech/singing/music 区分 → 修正 SongFormer 误判 | `pretrained_models/xukaituo/FireRedVAD/AED/` | GPU | AudioAnalyzer.vad |
| **Silero VAD** | 语音活动检测（FireRedVAD 降级备选） | `torch.hub.load('snakers4/silero-vad')` | GPU/CPU | AudioAnalyzer.vad |
| **EasyOCR** (ch_sim+en) | 视频帧文字识别（标题画面检测） | `easyocr` pip | GPU | TextFrameDetector |
| **MSAF** (0.1.80) | 音乐结构边界检测 (Foote/SF/OLDA) | `msaf` pip | CPU | `_detect_msaf_boundaries` |
| **ShazamIO** | 音频识曲（asyncio 异步） | `shazamio` pip | - | `lyric_subtitle.py` |
| **ShazamAPI** | 音频识曲（ShazamIO 降级备选） | `ShazamAPI` pip | - | `lyric_subtitle.py` |
| ~~ACRCloud~~ | ~~音频识曲~~ | ~~API~~ | - | ❌ 已弃用 |
| **豆包 API** | ASR 字幕生成 + ASS 烧录 | 云端 API | - | `enable_subtitle` |

### 5.4 模型加载链（全部懒加载）

```
用户点"开始处理"
  → LiveVideoProcessor.process_video()
    → AudioAnalyzer → FireRedVAD/AED（首次加载）
    → TextFrameDetector → EasyOCR（首次加载）
    → MSAF（import msaf）
    → SongFormer（首次加载）
      → MuQ-large
      → MusicFM25Hz
      → SongFormer.safetensors
    → Demucs htdemucs_ft（如 enable_demucs）
    → ShazamIO（识曲时按需）
```

### 5.5 Step 0 高级设置项

| 配置项 | 默认值 | 范围 | 说明 |
|--------|--------|------|------|
| 最小时长 | 8.0s | 8-20s | 片段最短持续时间 |
| 最大时长 | 15.0s | 8-20s | 片段最长持续时间 |
| 切割模式 | fast | fast/accurate | fast=关键帧对齐，accurate=逐帧精确 |
| 生成字幕 | False | - | 勾选后使用豆包 API 生成字幕 |
| 识曲优先级 | acrcloud,shazamio,shazam | - | 按顺序尝试识曲服务 |

### 5.6 Step 2 编辑功能

- **时间轴跳转**：点击任意片段按钮 → 视频跳转到该片段起始时间播放
- **标签编辑**：12 类标签可选（前奏/主歌/副歌/间奏/独奏/观众/讲话/演说/人群/尾奏/其他/静默）
- **时间微调**：可调整起止时间（0.1s 步进），相邻片段自动联动
- **重置修改**：一键恢复所有原始标签
