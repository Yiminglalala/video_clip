## 1. 目标
- 将 PRD 转换为决策完整的技术方案，明确接口、数据流、失败处理与测试要求。

## 2. 适用范围
- 所有会变更代码行为、配置行为、运行流程的需求项。

## 3. 输入与输出
- 输入：PRD 条目、现状代码、历史问题。
- 输出：可直接实现的技术方案与验收约束。

---

## 4. 当前主链路基线（截至当前仓库）
- UI 层：`app.py`（双 Tab：视频切片 / 字幕生成）
- 核心编排：`src/processor.py`
- 音频分析与分类：`src/audio_analyzer.py`
- SongFormer 封装：`src/songformer_analyzer.py`
- 媒体处理：`src/ffmpeg_processor.py`

## 5. 技术方案模板（必须）

### 5.1 需求映射
- **对应 PRD ID**：填写对应的PRD ID，如PRD-2026-001
- **用户价值**：描述给用户带来什么价值，如"演唱会视频中的观众合唱片段能被正确识别，提升用户体验"
- **技术目标**：描述技术目标，必须可量化，如"调整audience标签评分规则，合唱识别准确率从当前<30%提升到>70%"

### 5.2 改动点
- **模块 A**：描述模块和改动内容，如"src/audio_analyzer.py：调整audience标签评分规则"
- **模块 B**：描述模块和改动内容，如"src/processor.py：添加audience标签详细调试日志"
- **是否新增配置**：是/否，如果是，说明新增什么配置

### 5.3 接口与数据契约
- **输入参数**：描述输入参数变更，如"无新增参数变更"
- **输出结构**：描述输出结构变更，如"新增日志字段：[REFINE-DEBUG] audience/crowd/solo/chorus/verse得分"
- **错误码/错误信息**：描述错误码或错误信息变更，如"无"
- **向后兼容策略**：描述向后兼容性，如"完全向后兼容，配置可回退"

### 5.4 失败模式与回退
- **失败模式 1**：描述可能的失败模式，如"调整规则后主歌/副歌识别准确率下降>10%"
- **回退路径 1**：描述回退方案，如"回退到旧版本audio_analyzer.py和processor.py"
- **阻断条件**：描述什么情况下阻断发布，如"主歌/副歌识别准确率下降>15%"

### 5.5 观测与日志
- **新增日志字段**：描述新增的日志字段，如"[REFINE-DEBUG]，包含每个片段的所有标签得分"
- **run_summary 影响字段**：描述run_summary的影响，如"新增type_counter.audience字段统计"

### 5.6 测试要求
- **Smoke**：描述Smoke测试要求，如"测试视频2（周深-VX-q97643800 (17).mp4），验证45-56秒是否被识别为合唱"
- **Regression**：描述Regression测试要求，如"测试4个测试视频，验证主歌/副歌识别准确率保持不变"
- **性能**：描述性能测试要求，如"处理速度保持<10秒/5分钟视频"
- **验收阈值**：描述验收阈值，如"合唱识别准确率>70%，主歌/副歌识别准确率波动<5%"

## 6. 运行结果结构化产物（最小字段）
- `run_summary.json` 建议包含：
  - `run_id`
  - `timestamp`
  - `input_video`
  - `elapsed_sec`
  - `song_count`
  - `type_counter`
  - `unknown_rate`
  - `output_count`
  - `errors`
  - `pass`（是否达标）

## 7. 验收标准
- 每个技术变更必须能映射回 PRD 条目。
- 每个变更必须声明"输入输出、失败回退、测试要求"。
- 未更新 TECH_SPEC 的变更不得进入发布清单。

## 8. TECH_SPEC示例（可直接照抄）

### 示例1：优化合唱识别准确率（对应 PRD-2026-001）

#### 5.1 需求映射
- **对应 PRD ID**: PRD-2026-001
- **用户价值**: 演唱会视频中的观众合唱片段能被正确识别，提升用户体验
- **技术目标**: 调整audience标签评分规则，合唱识别准确率从<30%提升到>70%

#### 5.2 改动点
- **模块 A**: `src/audio_analyzer.py
  - 改动内容：调整audience标签评分规则
    - 入场条件：voice_ratio > 0.25 → 0.15
    - voice_ratio范围：0.30-0.85 → 0.15-0.95
    - beat_regularity要求：< 0.35 → < 0.75
    - harmonic_ratio减分项阈值：> 0.7 → > 0.75
- **模块 B**: `src/processor.py`
  - 改动内容：
    - 在_refine_songformer_segments_labels中添加audience标签详细调试日志
    - 降低audience标签检测阈值：5.0 → 4.0
- **是否新增配置**: 是，CONFIG.AUDIENCE_SCORE_THRESHOLD已在config.py中

#### 5.3 接口与数据契约
- **输入参数**: 无新增参数变更
- **输出结构**:
  - 新增日志字段：[REFINE-DEBUG] audience/crowd/solo/chorus/verse得分
- **错误码/错误信息**: 无
- **向后兼容策略**: 完全向后兼容，配置可回退

#### 5.4 失败模式与回退
- **失败模式 1**: 调整规则后主歌/副歌识别准确率下降>10%
- **回退路径 1**: 回退到旧版本audio_analyzer.py和processor.py
- **阻断条件**: 主歌/副歌识别准确率下降>15%

#### 5.5 观测与日志
- **新增日志字段**: [REFINE-DEBUG]，包含每个片段的所有标签得分
- **run_summary 影响字段**: 新增type_counter.audience字段统计

#### 5.6 测试要求
- **Smoke**: 测试视频2（周深-VX-q97643800 (17).mp4），验证45-56秒是否被识别为合唱
- **Regression**: 测试4个测试视频，验证主歌/副歌识别准确率保持不变
- **性能**: 处理速度保持<10秒/5分钟视频
- **验收阈值**: 合唱识别准确率>70%，主歌/副歌识别准确率波动<5%
