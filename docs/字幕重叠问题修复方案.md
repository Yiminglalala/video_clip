# 视频切片字幕重叠问题修复方案

## 📋 问题分析

### 问题现象
- 切片视频末尾经常包含下一段视频的起始字幕内容
- 手动缩短切片 0.2-0.5 秒可以暂时规避
- 多首歌曲混剪时更严重

### 根本原因

| 层面 | 问题位置 | 描述 |
|------|----------|------|
| 字幕筛选 | `_generate_subtitles_from_cached_asr:2375` | `if s_end > seg_start and s_start < seg_end:` 条件过宽松，只要有交集就会包含 |
| 字幕对齐 | `_expand_to_sentence_boundary` | 优先向句子边界扩展，而不是安全地在之前切断 |
| 安全边距 | 全局缺失 | 切片结束边界没有预留安全距离 |

## 🛡️ 解决方案：三层保护机制

### 第一层：字幕筛选安全保护
**文件**: `src/processor.py`

**修改**: `_generate_subtitles_from_cached_asr`

```python
# 安全边距
SAFETY_MARGIN = 0.15
safe_seg_end = seg_end - SAFETY_MARGIN

# 筛选策略：
# 1. 只包含完全在安全边界内的句子
# 2. 句子起始在边界内但结束超出时：只包含到安全边界
# 3. 句子起始在边界外但结束在边界内：部分包含
```

### 第二层：字幕对齐智能策略
**文件**: `src/subtitle_alignment.py`

**修改**: `_expand_to_sentence_boundary`

```python
# 对于结束边界，智能策略：
# 1. 优先寻找最近的、在时间点之前的句子结束点
# 2. 在句子结束点前增加安全边距
# 3. 如果没有，使用智能裁剪
```

### 第三层：视频切片安全边距
**文件**: `src/ffmpeg_processor.py`

**修改**: `cut_video`

```python
SAFETY_MARGIN = 0.15
safe_end_time = end_time - SAFETY_MARGIN
safe_start_time = start_time + SAFETY_MARGIN
```

## 📐 参数说明

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| `SAFETY_MARGIN` | 所有模块 | 0.15s | 安全边距，防止包含下一段内容 |
| `_BOUNDARY_EPSILON` | subtitle_alignment | 0.05s | 边界容差 |
| `_SENTENCE_GUARD_SHIFT` | subtitle_alignment | 4.0s | 句子保护扩展 |

## 📝 使用说明

### 无需额外配置
- 所有修改自动生效
- 三层保护机制默认开启
- 安全边距参数已优化

### 调整参数（可选）
如果需要调整安全边距，修改各模块中的 `SAFETY_MARGIN` 常量即可。

## 🔍 验证方法

1. 处理一段包含多首歌曲的视频
2. 检查每个切片的最后一帧字幕
3. 验证是否不再包含下一段视频的起始内容

## 📌 注意事项

- 安全边距会轻微缩短切片时长（约 0.15s），对内容影响很小
- 三层保护相互配合，效果最佳
- 建议先测试验证再大规模使用
