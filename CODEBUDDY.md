# AI 写作风格规范

本文件由 WorkBuddy 自动加载，AI 在本项目中的所有输出必须遵循以下规范。

## Output Contract

回答长度：
- 事实性问题：≤3句
- 解释性问题：≤1段，不分节
- 对比类问题：1-2句说明差异，不展开每一方

## Verbosity Controls

### 禁止

- 加粗标题分节
- 编号列举能一句话说清的内容
- 并列论点超过3个时未先检查是否可合并
- 罗列原始数据后再给结论（直接给结论，数据仅在被要求时提供）
- 转折句式「不是X而是Y」，直接陈述Y
- 反向排除论证：通过否定多个可能性间接导出结论；直接陈述因果链，例：A → B → C → D
- 引导句：「好的」「我来解释」「结论如下」
- 重复用户问题或复述背景
- 比喻和类比
- 主动提供未被要求的后续步骤
- 以「如果你需要/如果你要，我可以...」结尾
- 技术上限/范围用模糊限定词表达（如"远大于""取决于""较大"）；必须给出精确数值；若无把握，先联网查证再回答
- 忽略用户显式格式指令（如要求 ASCII 图时改用文字描述）
- 对比分析只列一侧（必须给出标准结构与异常结构的字段级并排对比）
- 段落间重复：后段不得用文字重述前段已用图或示例表达的内容
- 用模糊位置词替代具体值（如"更靠后"→必须用实际字节位置或行内容标注）
- 技术术语未定义直接使用；首次出现时给出操作级定义，格式：「术语 = 具体含义」，禁止用比喻替代

### 允许例外

- 内容为顺序步骤或结构层级（如解析流程、字段定义）时，允许编号列举；优先压缩至≤3条，硬上限9条；超过9条须先分组再列举
- 论点存在从属关系时，使用层级缩进表达；主论点顶格，子论点缩进一级，结论或最优解缩进标注；禁止把从属关系压平成同级列举

## 示例

反向排除 → 因果链：
- [冗余] 这种形态不是"字段组合变了"，也不是parser造成的，因为Trim()不会把行首裁掉，"写端尚未写完"不会稳定产生后半截。
- [简洁] 行首截断 → 读端在写操作完成前读取 → 获得尾残片。

数据罗列 → 分界点：
- [冗余] 19:01:38首次出现Fatal error，19:01:39开始后续全部变成 Connection must be valid and open
- [简洁] 19:01:39为分界点，之前正常插入，之后持续报错：Connection must be valid and open

结尾引导 → 直接结论：
- [冗余] 根因是套接字访问被拒绝...如果你要，我下一步可以做两件事之一：...
- [简洁] 根因是套接字访问被拒绝（192.168.5.12:3306）。检查：MySQL是否监听3306、防火墙是否放行、出站规则是否拦截pwsh.exe。

整条回复 → 层次清晰：
- 传导点2：...根因...
- 传导点3：打断方法有02条：
  - 01：...
  - 02：...
- 加强方案：01：... → ... → ...
- 守护方案：01：...

模糊上限 → 精确数值：
- [冗余] System.String 上限远大于 1024，实际取决于可用内存。
- [简洁] System.String 单对象上限 2 GB（约 10 亿字符），String.Length 类型为 Int32，理论计数上限 2,147,483,647；当前瓶颈是 VARCHAR(1024)，不是 PowerShell 类型声明。来源：Microsoft Learn / system-string, string.length

## 项目特定规范

- Streamlit widget key 用了 `key="xxx"` 会锁定 session_state，外部不可赋值 → 需要外部写值时不要用 key
- FFmpeg Windows `ass=` 滤镜会把含冒号路径解析为尺寸 → 必须 cd 到目录用相对路径
- Python raw string 单引号嵌套：`r'...\'` 中 `\'` 会关闭字符串 → 改用双引号包裹
- FunASR 词级时间戳是秒（不是 Whisper 的毫秒）
- 改完代码必须跑完整测试再交付
- 预处理位置是致命问题：Whisper/Fusion 走原始音频+轻量归一化，只有 FunASR 才做带通+预加重+压缩
- Python or 陷阱：`int(val or -1)` 当 val=0 时错误 fallback → 用 `int(val) if val is not None else -1`
